# 🚀 AutoISO - 分布式高可用自动封装与转存系统

![Version](https://img.shields.io/badge/version-v0.9.7-blue.svg)
![Python](https://img.shields.io/badge/python-3.8%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

AutoISO 是一个专为 PT (Private Tracker) 玩家和影视发烧友设计的**分布式自动化存储工作流系统**。它能够完美联动 qBittorrent、TMDB 以及 CloudDrive2，实现从“下载完成”到“生成 ISO 镜像”，再到“跨国无缝转存至云盘”的全自动、无人值守体验。

## ✨ 核心特性

- 🕸️ **分布式架构 (Master-Edge)**：以群晖 NAS 作为中央主控，支持挂载无数个海外 VPS 作为“边缘节点”。统一在 NAS 面板下发指令、查看进度、全局管理。
- 📦 **自动化 ISO 封装**：精准识别 qB 任务，针对多文件目录自动调用 `genisoimage` 进行底层无损封装，单文件则极速秒转。
- 🎬 **TMDB 智能刮削引擎**：内置自动化名称清洗与 TMDB 匹配算法，为枯燥的文件赋予精美的海报与影视元数据，支持 Web 端一键手动纠错与重新绑定。
- ☁️ **无缝对接 CloudDrive2**：针对 CD2 (FUSE) 挂载盘特性深度优化，彻底打通本地与云端的物理壁垒，支持跨国大文件稳定推送。
- 🛡️ **工业级防假死与自愈机制**：VPS 边缘节点采用“双线程高频心跳 + 断网自毁重生”架构，彻底告别 TCP 状态僵死与网络波动导致的节点失联。
- 📱 **Telegram 实时推送**：任务封装、上传、失败全流程精准推送，随时随地掌握系统动态。

## 🚀 部署与安装

### 1. 中央主控部署 (推荐在 NAS / 软路由上运行)

本项目提供开箱即用的 Flask Web 服务。请确保宿主机已安装 Python 3.8+ 及必要依赖。

```bash
# 克隆仓库
git clone [https://github.com/yourusername/AutoISO.git](https://github.com/yourusername/AutoISO.git)
cd AutoISO

# 安装依赖
pip install -r requirements.txt

# 运行主控服务
python3 app.py
```

> **注**：默认运行在 `0.0.0.0:5000`。首次访问将自动初始化默认管理员账号（账号：`admin`，密码：`admin123`），请登录后立即前往“系统设置”修改密码。

### 2. 边缘节点部署 (VPS 一键安装)

**完全无需手动配环境！**
1. 登录 AutoISO 主控 Web UI。
2. 进入左侧【边缘节点】菜单，点击 `添加节点` 并填写 VPS 本地 qB 信息及 CD2 挂载路径。
3. 保存后，点击表格右侧的绿色 **[部署]** 按钮。
4. 复制弹出的 Bash 部署代码，直接粘贴到你的 VPS SSH 终端执行即可。

> *部署脚本会自动安装运行环境、建立双线程身份凭证，并将其注册为 `systemd` 守护进程，实现开机自启与断网故障自愈。*

## ⚙️ 核心使用流

1. **打标签接管**：在任意 qBittorrent (NAS 或 VPS) 中，为下载完成的种子打上 `待封装` 标签。系统监控到后，将立即触发 TMDB 提前刮削，并在后台调用底层命令生成 `.iso` 文件。完成后 qB 标签将自动变为 `已封装`。
2. **全局调度 / 立即上传**：
   - **全局定时**：系统按照设置的 Cron 表达式（如 `0 2 * * *` 凌晨 2 点），自动向所有节点下发通行证进行批量上传。
   - **立即上传**：在 Web 端的“待上传队列”中手动点击“立即上传”，任务将获取极速特权，跳过定时规则立刻开始转存。
3. **差异化空间清理**：
   - **NAS 主控端**：由于本地空间通常较充裕，您可以在“系统设置”中自由开关**“上传后删除已封装文件”**选项，灵活决定是否保留本地备份。
   - **VPS 边缘节点**：受限于云服务器的高昂存储成本，VPS 在完成向网盘的转移后，会**强制自动清理**本地的临时 ISO 缓存，严防小盘爆满。

## ⚠️ 最佳实践与注意事项

**💡 强烈建议：为 VPS 节点分配专属的“空目录”作为网盘接收站**
在使用海外 VPS 节点配置 CD2 的底层挂载路径（`cd2_path`）时，**千万不要**将其直接挂载到包含海量老文件的庞大影视库目录中。强烈建议在 115 网盘中新建一个专门的空文件夹（例如：`/VPS_Transfer`），专门作为中转接收站。

* **原理解析（核心避坑）**：由于大多数 VPS 使用的是海外 IP，而 115 网盘对海外 IP 的 API 请求有着极其严格的**风控机制**。如果挂载的目录中包含海量文件，CloudDrive2 在底层同步时会频繁调用 115 的接口去扫描和比对目录树。**这种高频的 API 交互叠加海外 IP，极易触发 115 的风控雷达**，直接导致 VPS 网络假死、上传中断甚至账号限制。而使用**专属的空目录**，能将 API 调用量降到绝对的最低值，让 VPS 像个“透明人”一样只负责底层的数据传输，完美规避网盘风控！

## 🛠️ 常见问题 (FAQ)

**Q: 为什么在 VPS 上点击“立即上传”后，115 网盘里没有立刻出现文件？**
A: 这是正常的物理机制。VPS 收到立即上传指令后，脚本会瞬间将数十 GB 的文件移入 CloudDrive2 的本地 FUSE 缓存中（UI 会立刻变绿显示“已上传”）。随后，CD2 会在后台吃满你的 VPS 带宽，将文件通过跨国网络推向国内网盘。这个过程受限于实际网速，可能需要十几分钟至数小时。此时网盘端暂不显示文件，请耐心等待，不必焦虑。

**Q: 为什么重启主控 NAS 的 Docker 容器后，VPS 会在短时间内断联？**
A: 容器重启会重置宿主机的 TCP NAT 状态表，导致 VPS 原本的长连接（Keep-Alive）发送的数据包被默默丢弃（TCP 假死）。**不必担心**，本系统为 VPS 节点设计了“连续失联 60 秒触发系统级自尽”功能。Linux 的 `systemd` 会在 5 秒内将自尽的进程满血复活，重新发起全新的握手连接，实现对假死状态的无感自愈。

**Q: 如何查看 VPS 节点的真实后台干活进度？**
A: 登录对应 VPS 的 SSH 终端，执行 `journalctl -u autoiso-agent -f`，即可实时查看极其详尽的双线程运作、打包进度与跨国传输日志。
